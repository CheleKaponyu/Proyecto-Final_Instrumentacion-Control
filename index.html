Hallazgo clave

Cuando el motor DC se activaba, el Arduino se apagaba por demanda de corriente.
La solución fue incorporar un puente H L298N para aislar la etapa de potencia y mantener la estabilidad del sistema.

Proyecto y planteamiento

El proyecto se diseñó para monitorear variables ambientales con Arduino y accionar un elemento de respuesta de forma estable. La propuesta original incluía temperatura, humedad e iluminación; en la implementación física final se validó el comportamiento del sistema usando un motor DC como actuador representativo y una etapa de potencia adecuada para evitar fallas eléctricas.

Descripción resumida

Este prototipo integra sensado, procesamiento y actuación en una sola plataforma. El Arduino recibe lecturas del entorno, aplica una lógica simple por rangos y controla la velocidad del actuador mediante PWM. Durante el montaje se identificó un problema real de alimentación, lo que permitió reforzar el diseño con mejores prácticas de potencia.

Sensado ambiental Control por rangos PWM Pruebas con multímetro Monitor serial

¿Qué problema resolvió?

  • Lectura de temperatura con el DHT22 para tomar decisiones de control.
  • Visualización de datos en tiempo real desde el monitor serial y el serial plotter.
  • Corrección del apagado del Arduino al activar el motor DC.
  • Validación del sistema con mediciones eléctricas y fuente variable ajustada a 7 V.

Objetivos del proyecto

El proyecto buscó diseñar e implementar un sistema funcional, coherente con la propuesta inicial y capaz de demostrar en la práctica cómo se combinan sensores, lógica de control y una etapa de potencia en un montaje real.

01

Objetivo general

Diseñar e implementar un sistema de control ambiental basado en Arduino Uno, capaz de medir variables del entorno y accionar un elemento de respuesta mediante una lógica simple y estable.

02

Integración de sensores

Conectar el DHT22 y la LDR al Arduino para obtener lecturas del entorno y usarlas como base del comportamiento del sistema.

03

Etapa de potencia

Resolver los problemas de alimentación del actuador mediante una etapa adecuada, validando el prototipo con pruebas prácticas y mediciones de voltaje.

Arquitectura y funcionamiento

El sistema sigue una secuencia sencilla: lectura del sensor, comparación con límites establecidos y actuación del motor DC por PWM. Esta estructura permitió demostrar el principio de control ambiental con una implementación clara y funcional.

Flujo del sistema

Paso 1

Lectura del DHT22

El Arduino toma la temperatura y verifica que la lectura sea válida antes de continuar.

Paso 2

Comparación por rangos

La temperatura se compara con tres límites: 16 °C, 20 °C y 23 °C.

Paso 3

Control PWM

Según el rango detectado, el sistema asigna 0 %, 40 %, 65 % o 100 % de velocidad al motor.

Paso 4

Visualización

Los datos se envían al monitor serial y al serial plotter para observar temperatura, límites y porcentaje de velocidad.

Código de programación

Este apartado incluye el programa utilizado para leer la temperatura, definir el porcentaje de velocidad del motor según rangos y enviar los datos al monitor serial.

Arduino IDE · Control de temperatura con DHT22 + L298N 
#include <DHT.h>

// ----------- DHT22 -----------
#define DHTPIN 2
#define DHTTYPE DHT22
DHT dht(DHTPIN, DHTTYPE);

// ----------- L298N -----------
int ENA = 9;
int IN1 = 7;
int IN2 = 6;

void setup() {
  Serial.begin(9600);
  dht.begin();

  pinMode(ENA, OUTPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);

  // Dirección fija
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
}

void loop() {

  float temp = dht.readTemperature();

  if (isnan(temp)) {
    return;
  }

  int velocidadPWM = 0;
  float porcentaje = 0;

  // -------- CONTROL POR RANGOS --------
  if (temp <= 16) {
    porcentaje = 0;
  }
  else if (temp > 16 && temp <= 20) {
    porcentaje = 40;
  }
  else if (temp > 20 && temp <= 23) {
    porcentaje = 65;
  }
  else {
    porcentaje = 100;
  }

  // Convertir a PWM
  velocidadPWM = (porcentaje / 100.0) * 255;

  analogWrite(ENA, velocidadPWM);

  // -------- SERIAL PLOTTER --------
  Serial.print(temp);
  Serial.print(" ");
  Serial.print(16);
  Serial.print(" ");
  Serial.print(20);
  Serial.print(" ");
  Serial.print(23);
  Serial.print(" ");
  Serial.println(porcentaje);

  delay(1000);
}
¿Qué hace este código?

Inicializa el sensor DHT22 en el pin 2 y el driver L298N con los pines ENA, IN1 e IN2. Luego lee la temperatura, verifica si el dato es válido y aplica una lógica por rangos: 0 % si la temperatura es menor o igual a 16 °C, 40 % entre 16 y 20 °C, 65 % entre 20 y 23 °C, y 100 % cuando supera 23 °C. Después convierte ese porcentaje a una señal PWM para controlar la velocidad del motor y envía al serial la temperatura, los límites de referencia y el porcentaje de velocidad para su visualización en el monitor serial y el serial plotter.
